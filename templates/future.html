Looking Forward

Scaling Framework

In the future, we would like to build a more robust framework to scrape from Twitter large amounts of images from different parts of the world. During this hackathon, we were limited by the number of API calls we could make, which we would be able to work around given more time. We would also extend our framework to make use of our Microsoft Azure Machine Learning endpoints to batch process the data mined from the Twitter images.

Improving Model Accuracy

The main constraint in developing a good machine learning algorithm was time. There are many ways in which we could have possibly improved the classifier. One of our main problems was that there was a bias inherent in our model, which would be improved by gathering a much larger dataset. Although we developed an infrastructure to collect and process data, we needed more time to create a sizable dataset as we were limited by API request limits and time. Another improvement we could have would be to use a more sophisticated model to classify the data (kernelized SVM, neural networks, etc.), as the high bias and low variance indicated underfitting of data. Additionally, if enough data were to be collected, we could use unsupervised learning via k-means clustering to create clusters and then label them through "semi-supervised" learning by labeling one data point in each cluster.

Add Features to Map

Lastly, we would add more functionality to the map. As mentioned before, the framework could be scaled to analyze even images per location and zoom into specific locations within each state. We could also display the top tweets and images per region that contributed to the different emotions of that region, allowing users of the app to get a sense of current events around the country.
