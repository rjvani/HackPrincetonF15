Looking Forward

Scaling Framework

In the future, we would like to make a more robust framework that can scrape more images from different parts of the globe from Twitter. In this hackathon, we were limited by API calls in a condensed frametime, but this is not a limitation outside of this competition. We would also want to make use of the Microsoft Azure Machine Learning endpoints to batch process the data we mined from the images.

Improving Algorithm Accuracy

Time was definitely one of the main constraints in this machine learning problem. There are many ways in which the classifier could have been improved. One improvement would be the creation of more data. Although we made an infrastructure to generate such a dataset, more time would be needed to populate it. Further, a more sophisticated algorithm could be used to classify the data (such as a kernelized SVM, neural networks), as the bias indicated some underfitting. Lastly, if enough data were collected, we could use k-means clustering, a popular unsupervised learning algorithm, to classify clusters and then label them through "semi-supervised" learning by labeling one data point in the cluster.
